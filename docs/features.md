# PromptRelay 機能候補

チェックを入れた機能を採用とする。

## セッション管理

- [x] 対象ディレクトリを指定して Claude Code / Codex セッションを子プロセスとして起動
- [x] 現在アクティブなセッション一覧の表示（PID、対象ディレクトリ、状態、経過時間）
- [x] 個別 or 一括でセッションを終了
- [x] session_id を使って過去のセッションコンテキストを引き継いで再開
- [x] 複数セッションをタブ的に切り替えて操作
- [x] セッションごとに Claude Code / Codex を選択可能
- [x] 複数ディレクトリに対して同時にセッションを起動・実行

## プロンプトキュー

- [x] テキストエディタでプロンプトを書いてキューに追加
  - Tsugi UI上でプロンプトを入力して「キューに追加」ボタンが欲しい
- [x] 登録済みプロンプトの一覧表示（順序・状態）
- [x] ドラッグ&ドロップでプロンプトの実行順を変更
- [x] 実行中のキューに新しいプロンプトを途中挿入
- [x] キュー内の未実行プロンプトをその場で編集
- [x] キューからプロンプトを削除
- [x] 複数プロンプトをまとめて登録（改行区切り、ファイルからインポートなど）

## 実行制御

- [x] キュー内のプロンプトを順次自動で実行
- [x] 各プロンプトの実行前にユーザー確認を挟む手動実行モード
- [x] 実行中でも自動/手動をモード切り替え可能
- [x] 実行を一時停止して再開
- [x] 現在のプロンプトをスキップして次へ
- [x] 実行中のプロンプトを強制中断
- [x] 失敗したプロンプトをリトライ
- [x] プロンプトごとに最大実行時間を設定（タイムアウト）

## テンプレート・変数

- [ ] プロンプト内に `{{変数名}}` を埋め込み、実行時に値を注入
- [ ] `{{prev.output}}` のように前のステップの出力をテンプレート変数として使用
- [ ] `{{file:path/to/file}}` でローカルファイルの内容をプロンプトに埋め込む
- [ ] `{{env:VAR_NAME}}` で環境変数を参照
- [ ] 実行時に未定義の変数があればユーザーに入力を求める（対話的変数入力）

## フロー（再利用可能なプロンプトチェーン）

- [x] 複数プロンプトの順序・変数・条件をまとめて名前付きフローとして保存
- [x] 保存済みフローをディレクトリを指定して実行
- [ ] フロー実行時に引数を渡せる（パラメータ化）
- [x] JSON/YAML でフローをインポート/エクスポート
- [ ] フロー定義の変更履歴（バージョン管理）

## 条件分岐・制御フロー

- [x] 前ステップの出力に応じて次のプロンプトを分岐
  - 作業セッション(A)の出力を、判定用セッション(B)に自動で渡して判定させる
  - ユーザーはフローの分岐ノードに「判定プロンプト」を書くだけ（例: テストが通っているか yes/no で判定）
  - Tsugi が裏で出力の受け渡しと判定結果の取得を行い、結果に応じて次のプロンプトを分岐
  - テンプレート変数は不要（同一セッション内はコンテキスト自動保持、別セッションへの受け渡しは分岐ノードが処理）
- [x] 条件を満たすまでプロンプトを繰り返し実行（ループ）
- [x] 出力が期待パターンに合致するかチェック、失敗時にリトライや分岐（出力バリデーション）
- [x] 特定ステップの後にユーザー承認を必須にする（承認ゲート）

## モニタリング・出力

- [x] セッションの stdout/stderr をリアルタイム表示
- [ ] コードブロックやdiff出力のシンタックスハイライト
- [x] Claude Code がファイル読み書き・コマンド実行した際のツール使用の可視化
- [x] プロンプトごと / セッションごとのトークン使用量表示
- [x] トークン使用量からのコスト見積もり表示

## ログ・履歴

- [x] 過去の実行一覧（日時、ディレクトリ、フロー名、成否）
- [x] 各ステップの入力プロンプトと出力を永続化
  - フロー定義・プロンプトはファイル（JSON/YAML）、実行ログ・出力はSQLite
- [x] ディレクトリ名、日付、キーワードで履歴を検索・フィルタ
- [x] 過去の実行をそのまま or 編集して再実行
- [x] 実行ログをファイルにエクスポート

## プロジェクト管理

- [x] よく使うディレクトリをプロジェクトとして登録・管理
- [ ] プロジェクトごとにデフォルトCLI、フロー、変数を設定
- [x] 直近で操作したディレクトリの一覧（最近使ったディレクトリ）
- [ ] プロジェクトをカテゴリやタグで分類（グルーピング）

## Hooks・拡張

- [ ] プロンプト実行前にシェルコマンドを実行（プリフック、例: git pull）
- [ ] プロンプト実行後にシェルコマンドを実行（ポストフック、例: テスト実行、lint）
- [ ] フックの成否で次のステップを制御
- [ ] Claude Code の Hooks イベント（PostToolUse 等）を受信して表示・制御
- [ ] Tsugi 自体を MCP サーバーとして公開、またはカスタム MCP ツールを追加

## Git 連携

- [ ] セッション実行前後の git diff を表示
- [ ] 各ステップ完了後に自動で git commit
- [x] フロー実行時に git worktree で隔離された作業環境を自動作成
- [ ] 実行結果が期待と異なる場合に git reset でロールバック

## 通知

- [x] フロー完了時にシステム通知
- [x] 失敗時にエラー通知
- [x] 手動モードで次のプロンプト実行待ちの承認待ち通知
  - 質問が来たときも通知してほしい

## 設定

- [ ] claude / codex の実行パスを設定
- [x] デフォルトの CLI ツール、実行モード、タイムアウト等の設定
- [x] 主要操作のキーボードショートカット
