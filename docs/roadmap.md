# PromptRelay 開発ロードマップ

各マイルストーンで完動品としてリリース可能な状態を維持します。

## 依存関係

```text
v0.1 セッション起動・プロンプト送信
 ↓
v0.2 プロンプトキュー・自動実行
 ↓
v0.3 実行制御（手動モード・一時停止・リトライ等）
 ↓
v0.4 複数セッション・プロジェクト管理
 ↓
v0.5 モニタリング（ツール可視化・トークン・コスト）
 ↓
v0.6 永続化・履歴（SQLite）
 ↓
v0.7 フロー（保存・実行・インポート/エクスポート）
 ↓
v0.8 条件分岐・ループ・承認ゲート
 ↓
v0.9 Git worktree・通知・設定
```

## アーキテクチャ方針

```tree
Tsugi (Tauri App)
├── Frontend (React + TypeScript)
│   ├── SessionView    セッション起動・タブ切り替え
│   ├── QueueView      プロンプト入力・キュー管理
│   ├── OutputView     リアルタイム出力・ツール可視化
│   ├── FlowEditor     フロー定義・条件分岐
│   └── HistoryView    実行履歴・検索
├── Backend (Rust / Tauri Commands)
│   ├── SessionManager 子プロセスの起動・管理・stdin/stdout
│   ├── QueueEngine    キュー管理・実行制御・モード切替
│   ├── FlowEngine     フロー定義の読込・条件分岐・ループ
│   ├── Persistence    SQLite（実行ログ・出力）
│   └── Notifier       システム通知
└── CLI Adapters
    ├── ClaudeCodeAdapter  claude -p / -r / stream-json
    └── CodexAdapter       codex exec --json
```

### 設計判断

- **CLI Adapter trait**: Claude Code と Codex の差異を吸収するインターフェース。将来的な CLI ツール追加にも対応可能
- **子プロセスモデル**: 各セッションは CLI ツールの子プロセスとして起動。stdin/stdout でやりとり
- **セッション継続**: `claude -p -r <session_id>` による会話コンテキストの自動保持。テンプレート変数は不要
- **判定用セッション**: 条件分岐時は使い捨てセッションで判定。作業セッションのコンテキストは汚さない
- **永続化の分離**: フロー定義はファイル（JSON/YAML）、実行ログはSQLite
- **Tauri Commands**: フロントエンドとバックエンドの通信は Tauri の invoke 経由
